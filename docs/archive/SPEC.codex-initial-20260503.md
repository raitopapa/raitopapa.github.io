## 1. 現状把握

### 1.1 現在のプロダクト

静的 HTML + Vanilla JavaScript + PWA の GitHub Pages サイト。

- `index.html`: トップページ
- `diagnosis.html`: 病害虫リスク診断
- `pruning.html`: 剪定適期カレンダー
- `checklist.html`: 健全度チェックリスト
- `traq.html`: ISA TRAQ 風リスク評価
- `ctla.html`: CTLA 風価値算定 + i-Tree 風生態系サービス推計
- `survey.html`: GPS/地図/写真/CSV/GeoJSON/履歴つき現地調査
- `privacy.html`, `contact.html`, `manifest.json`, `sw.js`

強み:

- 日本語の樹木管理ツールとして、無料・即利用・ログイン不要の入口が明確。
- フィールド作業向けに GPS、地図、CSV/GeoJSON、オフライン、写真保存まで到達している。
- TRAQ、CTLA、i-Tree など専門家向けキーワードを押さえている。
- PWA、AdSense、OGP、JSON-LD、sitemap、privacy など公開サイトの最低限はある。

弱み:

- 全ページが巨大な単一 HTML で、共通 CSS/JS/データ定義が重複。今後の品質保証と多人数開発に不向き。
- `README.md` が文字化けして見える環境があり、開発者引き継ぎ品質が低い。
- `index.html` の CSS に明確な構文欠損がある。`.hero` セレクタが欠落し、行 64 付近で `.lang-btn:hover { ... } background: ... }` になっている。
- トップページの表示数が「3 専門ツール」なのに実際は 6 ツール構成。価値訴求と導線が不一致。
- `manifest.json` に存在しない `icons/screenshot-mobile.png` が指定されている。
- `sw.js` のコメント/ログは v1.3、キャッシュ名は `arborist-tools-v2.4` でバージョン体系が不整合。
- AdSense プレースホルダーが多いが、実広告コンポーネント、計測、同意管理、配置方針が未整理。
- `contact.html` は送信 UI があるが、本番送信先は未実装コメントのみ。リード獲得経路として機能していない。
- 「監修」「準拠」「法的」「査定」表現が強い。専門資格・出典・免責・計算根拠の正確性を精査しないと信用毀損リスクがある。
- CTA が無料利用中心で、サブスクリプション、法人導入、相談、資料請求などの収益導線がない。

## 2. 海外ベンチマークからの示唆

参考にした代表例:

- TreePlotter by PlanIT Geo: GIS 台帳、ダッシュボード、レポート、作業履歴、クエリ、地図共有、API 連携、オフライン収集、アドオン課金。
- i-Tree: 樹木構造と環境/経済価値を数値化し、意思決定・優先順位付け・説明責任に使う無料ツール群。
- MyTreesApp: 月額/年額サブスク、顧客・樹木管理、見積、作業票、安全装備管理、写真付き点検、クラウド保存、リソース教材。
- ArborNote: CRM、精密マッピング、提案書、スケジュール、作業指示、ジョブ原価、決済、QuickBooks 連携など、事業運営まで広げた高単価 SaaS。
- Treez などモバイルアプリ: オフライン、写真、危険度評価、DOCX/XLSX/画像レポート、多言語対応。

ベストプラクティス:

- 無料ツールは集客、プロ用台帳/レポート/案件管理は課金、という分離。
- 収益化の中心は広告ではなく、保存・共有・帳票・チーム・業務管理・クラウド同期。
- 地図 UI は単なる記録で終わらせず、フィルタ、色分け、履歴、優先順位、共有 URL、レポートに接続する。
- 専門ツールは「診断っぽい結果」より「監査可能な根拠」「入力履歴」「PDF/DOCX/XLSX」「免責」「専門家確認」が重要。
- 無料/低価格アプリは月額 10 EUR 前後、高機能 B2B SaaS は年額または 1 ユーザー月額 200 USD 級まで幅がある。日本版は段階課金が現実的。

## 3. 目標ポジショニング

短期:

- 日本の樹木管理・現地調査に特化した無料 PWA ツール集。
- 検索流入、SNS 共有、専門家の試用を増やす。
- AdSense と問い合わせを最低限機能させる。

中期:

- 個人樹木医・造園業・自治体委託業者向けの「現地調査台帳 + 帳票作成 + 写真管理」プロダクト。
- サブスクはクラウド同期、帳票テンプレート、案件/顧客管理、複数端末、チーム共有を課金対象にする。

長期:

- 日本版 TreePlotter/MyTreesApp の軽量版。
- 自治体、学校、寺社、マンション管理組合、造園会社向けに台帳・点検履歴・リスク説明資料を提供。

## 4. 課金設計

無料枠:

- 病害虫診断、剪定カレンダー、健全度チェック。
- 現地調査のローカル保存、CSV/GeoJSON 出力。
- AdSense 表示あり。

Pro 個人 月額 980-1,480 円:

- 広告非表示。
- PDF/DOCX/XLSX 帳票テンプレート。
- QR ラベル出力。
- 写真圧縮/EXIF 読み取り。
- 履歴比較と再点検アラート。

Business 月額 3,980-9,800 円:

- 顧客/案件管理。
- クラウド同期。
- 共有リンク。
- チーム権限。
- ブランド入り報告書。
- 見積/作業指示。

Enterprise 個別見積:

- 自治体/法人台帳インポート。
- 権限管理。
- 専用帳票。
- API/CSV 定期連携。
- 導入支援。

広告方針:

- 診断前後、結果下、記事型コンテンツに限定。
- 現地調査画面、入力中、緊急リスク表示中は広告を抑制。
- Pro の最初の価値は広告非表示ではなく、帳票・履歴・共有・クラウド同期。

## 5. 実装前の大方針

1. 先に壊れている表示と信頼性リスクを直す。
2. 次に情報設計、計測、問い合わせ、広告の収益土台を作る。
3. その後、サブスクに値する Pro 機能を作る。
4. 巨大 HTML のまま機能追加し続けない。段階的に共通 CSS/JS/データへ分離する。
5. 専門性表現は「準拠」ではなく「参照」「参考推計」「専門家確認が必要」を基本にする。正式資格・権利関係を確認できたものだけ強く言う。

受け入れ条件:

## 7. Sonnet/Claude/Codex 分担案

競合回避ルール:

- 仕様・計画は `docs/` に集約する。
- 同じ HTML を同時に編集しない。
- 1 タスク 1 ブランチまたは 1 ワークツリー。
- 作業開始時に `docs/SPEC.md` の該当 TASK ID を宣言する。
- 完了時に「変更ファイル」「確認方法」「残リスク」を `docs/HANDOFF_LOG.md` に追記する。

初期分担:

- Claude: `TASK-P0-004`, `TASK-P1-003`, `TASK-P1-004` の調査・文言・計測仕様。
